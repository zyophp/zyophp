# Guia de Contribuição

Obrigado por se interessar em contribuir para o ZyoPHP! Este é um projeto focado em simplicidade e boas práticas.

## Como Contribuir

1. **Relate Bugs**: Abra uma Issue descrevendo o problema e como reproduzi-lo.
2. **Sugira Melhorias**: Se tiver uma ideia legal, abra uma Issue para discutirmos.
3. **Envie Pull Requests**:
   - Faça um Fork do projeto.
   - Crie uma branch para sua nova feature (`git checkout -b feature/minha-feature`).
   - Siga os padrões de código (PSR-12).
   - Adicione testes se possível.
   - Envie o PR descrevendo suas mudanças.

## Padrões de Desenvolvimento

- **PSR-4**: Para autoloading.
- **PSR-12**: Para estilo de código PHP.
- **MVC**: Mantenha a separação entre lógica e apresentação.
- **Testes**: Use PHPUnit para garantir que as mudanças não quebrem o Core.

## Estrutura das Branches

- `main`: Versão estável.
- `develop`: Próximas mudanças.

## Perguntas?

Se tiver dúvidas sobre a arquitetura, consulte primeiro a [documentação detalhada](docs/README.md).
